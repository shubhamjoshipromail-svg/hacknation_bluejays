# Team Agent Rules

- Read README.md before doing anything.
- Never work directly on the main branch.
- Before editing files, pull the latest main and create a new feature branch.
- Use a branch name describing the assigned task, such as feat/frontend, feat/agent, feat/data, or fix/upload.
- Work only on the task assigned by the user.
- Do not edit or rewrite another teammate's work unless explicitly asked.
- Do not make large unrelated refactors.
- Do not push directly to main.
- Do not force-push main.
- Do not merge your own Pull Request.
- Do not delete branches or discard uncommitted work.
- Do not commit API keys, tokens, passwords, or .env files.
- Keep commits small and clearly described.
- Before finishing, pull the newest main, check for conflicts, and test the affected work.
- Push only the feature branch and open a Pull Request into main.
- In the final response, report the branch, files changed, tests run, and any conflicts or teammate follow-up needed.
- If the assigned branch, task, or owned files are unclear, stop and ask instead of guessing.
